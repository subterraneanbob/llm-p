# llm-p

Серверное приложения на FastAPI, предоставляющего защищённый API для взаимодействия с большой языковой моделью (LLM) через сервис OpenRouter.

## Установка

Загружаем код приложения:

```shell
git clone https://github.com/subterraneanbob/llm-p
```

Переходим в директорию с приложением:

```shell
cd llm-p
```
> N.B. В дальнейшем предполагается, что все команды выполняются из этой директории.

Для установки приложения потребуется пакетный менеджер uv. Для этого нужно следовать [инструкции по установке uv в систему](https://docs.astral.sh/uv/getting-started/installation/). После этого переходим к установке зависимостей.

### Установка зависимостей

Эта команда создаёт виртуальное окружение (если ещё не было создано) и устанавливает зависимости, необходимые для работы приложения:

```shell
uv sync
```

Другой способ: требует активное виртуальное окружение (и не сработает на Windows, т.к. использует process substitution).

Создаём виртуальное окружение:

```shell
uv venv
```

Активируем его:

```shell
source .venv/bin/activate
```

Устанавливаем зависимости:

```shell
uv pip install -r <(uv pip compile pyproject.toml)
```

## Настройка приложения перед запуском

Настройка осуществляется с помощью файла `.env` и переменных окружения, причём переменные окружения имеют приоритет, если используются оба варианта одновременно. Названия переменных можно посмотреть в файле `.env.example`.

### Пример настройки

Делаем копию файла-примера `.env.example`:

```shell
cp .env.example .env
```

Для нормальной работы приложения точно потребуется изменить несколько настроек из файла-примера:
- `JWT_SECRET` - секретный ключ для генерации JWT
- `OPENROUTER_API_KEY` - ключ доступа к API OpenRouter.ai

Остальные настройки можно изменять по желанию (их описание есть в самом файле).

Cгенерируем cекретный ключ `JWT_SECRET`:

```shell
# На любой платформе, где установлен OpenSSL
openssl rand -base64 32
# PowerShell
$bytes = [byte[]]::new(32); [System.Security.Cryptography.RNGCryptoServiceProvider]::Create().GetBytes($bytes); [Convert]::ToBase64String($bytes)
```

Для получения ключа доступа `OPENROUTER_API_KEY` требуется зарегистрироваться на платформе [OpenRouter](https://openrouter.ai) и создать ключ доступа на странице [API keys](https://openrouter.ai/workspaces/default/keys).

## Запуск приложения

Для запуска приложения выполняем команду:

```shell
uv run uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

Как только приложение вывело сообщение о готовности:

```
INFO:     Application startup complete.
```

можно приступать к работе. Так как приложение на имеет собственного UI, то можно сразу перейти к `Swagger UI` для выполнения запросов к API. В браузере переходим к http://0.0.0.0:8000/docs (если не сработало, то пробуем http://localhost:8000/docs или http://127.0.0.1:8000/docs).

## Рабочий процесс

1. Регистрация с помощью `POST /auth/register` - указываем почту и пароль.
2. Для авторизации используем кнопку **Authorize** наверху страницы; указываем там те же данные, что и на предыдущем шаге.
3. Для отправки запроса к LLM используем `POST /chat`. Указываем текст запроса к LLM (prompt). Опционально, можно указать системную инструкцию (system), количество сообщений (max_history), которые нужно включить в запрос и "температуру" (temperature, степень креативности модели). Этот метод можно использовать столько раз, сколько требуется для "общения" с языковой моделью.
4. Для получения всей истории переписки пользователя и LLM, используем `GET /chat/history`.
5. Для удаления истории пользователя применяется `DELETE /chat/history`.
